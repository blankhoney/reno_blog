## Context

Reno Blog is the public implementation of Personal Site Lab, a long-lived personal interface for writing, notes, project case studies, and controlled experiments.

## Problem

The site needs to stay easy for an agent to extend without drifting into a CMS, database-backed app, or runtime AI product.

## Goal

Build a bilingual static foundation that can publish real content, expose structured metadata, and leave room for future AI Q&A without adding backend complexity now.

## Process

The first iteration established Astro routes, MDX content collections, translation pairs, tests, Docker packaging, and GitHub Actions before adding larger experience work.

## Architecture

Astro renders static pages from typed content and small TypeScript data modules. React is reserved for isolated islands, while important reading and project content remains available as HTML.

## Outcome

The project now has bilingual routes, public content discovery, CI/CD, Docker packaging, and a clear path for Reader Shell, Library, content relations, and ProjectOrbit.

## Learnings

Static-first architecture keeps the site inspectable. The main risk is not missing features, but allowing experiments to leak into the core reading experience.
